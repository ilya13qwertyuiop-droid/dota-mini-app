## Часть 1 — Квиз по позициям

### Как запускается

- Кнопка «Подобрать позицию» на главной или «Начать тест» на странице квизов → `startPositionQuiz()`
- При старте: `currentQuestion = 0`, `scores = {pos1:0, pos2:0, pos3:0, pos4:0, pos5:0}`, `selectedAnswersHistory = []`

### Логика подсчёта результата

Функции: `getPositionResult()`, `getPurePosition()`, `showResult()`

#### Шаг 1 — Проверка чистых паттернов

Перед подсчётом очков проверяется, совпадают ли выбранные 5 ответов (по их ID) с одним из жёстко заданных паттернов.
Если совпадает → результат фиксированный, очки не смотрятся вообще.

#### Шаг 2 — Подсчёт по очкам (если паттерн не совпал)

Все 5 ответов суммируют свои очки в `scores = {pos1, pos2, pos3, pos4, pos5}`.
Позиции сортируются по убыванию суммы.

```
mainPos  = позиция с наибольшей суммой (первое место)
extraPos = позиция на втором месте
```

#### Шаг 3 — Формирование ключа для описания

| Тип результата | Ключ                    | Пример           |
|----------------|-------------------------|------------------|
| Чистый паттерн | `<pos>_pure`            | `pos3_pure`      |
| По очкам       | `<mainPos>_<extraPos>`  | `pos1_pos2`      |

Ключ используется для выбора текста-описания из объекта `positionDescriptions` (20 текстов для комбинаций + 4 для чистых).

#### Особенность: нет паттерна для «чистого pos4»

Паттерн определён только для pos1, pos2, pos3, pos5.
Если пользователь явно Pos 4 — он получит pos4 через очки, но с `extraPos` (например, `pos4_pos5`), а не через `_pure`.

### Как работает кнопка «Назад»

Функция `goBackInPositionQuiz()`:
1. Уменьшает `currentQuestion` на 1.
2. **Вычитает** очки последнего ответа из `scores` (полноценный откат).
3. Удаляет последний элемент из `selectedAnswersHistory`.
4. Показывает предыдущий вопрос.

## Часть 2 — Квиз по героям

### Откуда берутся герои

Герои загружены в 4 отдельных JS-файлах, подключённых в `index.html` до `script.js`:

| Файл                | Переменная          | Позиция                         |
|---------------------|---------------------|---------------------------------|
| `heroes-carry.js`   | `window.heroCarryData`  | Pos 1 — Керри               |
| `heroes-mid.js`     | `window.heroMidData`    | Pos 2 — Мидер               |
| `heroes-offlane.js` | `window.heroOfflaneData`| Pos 3 — Хардлейнер          |
| `heroes-pos45.js`   | `window.heroPos4Data`   | Pos 4 — Роумер              |
| `heroes-pos45.js`   | `window.heroPos5Data`   | Pos 5 — Фулл-саппорт        |

Каждый файл содержит два поля: `questions` (список вопросов для этой позиции) и `heroes` (список героев).

### Структура героя

**Для Pos 1 (Керри) и Pos 2 (Мидер)** — теги как массив строк:
```js
{ name: "Anti-Mage", tags: ["lategame", "melee", "splitpush", ...], difficulty: "hard" }
```

**Для Pos 3, 4, 5** — теги как объект с числовыми весами:
```js
{ name: "Lion", tags: { from_level: 1, from_control: 1, from_initiation: 1 }, difficulty: "easy" }
```

### Вопросы квиза по героям

Вопросы разные для каждой позиции (берутся из `questions` в соответствующем файле).
Каждый вариант ответа содержит список `tags`, которые добавляются в пул выбранных тегов игрока.

**Пример — Pos 1 (Керри), 5 вопросов:**
1. На каком этапе игры ты хочешь быть максимально полезен? → теги: `midgame`/`tempo`, `lategame`, `lategame`+`superlate`+`greedy`
2. Какой тип атаки тебе нравится? → теги: `melee` / `ranged`
3. Какой тип урона? → теги: `burst` / `sustained`
4. Как влиять на карту? → теги: `splitpush`+`map_pressure` / `teamfight` / `pickoff`+`map_pressure`
5. Насколько сложного героя готов освоить? → теги: `easy` / `medium` / `hard`

**Pos 4 (Роумер), 4 вопроса:**
1. От чего зависеть в игре? → `from_level` / `from_items`
2. Что делать в драках? → `from_control` / `from_damage` / `from_save`
3. Сложность? → `hard` / `easy`
4. Роль в файте? → `from_initiation` / `from_counterinitiation` / `from_position`

### Алгоритм подбора топ-5 героев (`calculateTopHeroes`)

```
Для каждого героя в списке позиции:
  score = 0

  Для каждого выбранного тега пользователя (кроме easy/medium/hard):
    Если hero.tags — массив (pos1/pos2):
      если tag входит в массив → score += 1
    Если hero.tags — объект (pos3/4/5):
      если hero.tags[tag] существует → score += hero.tags[tag]  (вес)

  Если выбранная сложность совпадает с hero.difficulty:
    score += 1.5  (бонус за сложность)

  Только для Pos 1 (керри), дополнительный бонус melee/ranged:
    Если игрок выбрал melee И герой только melee → score += 0.75
    Если игрок выбрал ranged И герой только ranged → score += 0.75

→ Сортировка по score убыванию → топ 5–6 героев
```

### Как считается «% совпадения»

Диапазон нормализуется по реальному разбросу среди топ-5:

```
normalized = (hero.score - min_score) / (max_score - min_score)
matchPercent = round(60 + normalized * 40)   → диапазон 60–100%
```

Если у всех героев одинаковый score → всем показывается фиксированные 75%.

| matchPercent | Рамка карточки  |
|--------------|-----------------|
| ≥ 90%        | Золотая         |
| 75–89%       | Серебряная      |
| < 75%        | Бронзовая       |
